# v0.7.0 — Scope model, unified index, and structured injection

**Status:** Design — awaiting implementation plan
**Date:** 2026-04-18
**Target release:** mnemo v0.7.0
**Supersedes:** none. Builds on v0.5.2 (project filter) + v0.6.2 (recall suite + rule-activation index).

---

## Background

The v0.6.2 architecture review surfaced two critical issues and three warnings in how mnemo differentiates project-specific vs global rules:

- **C1 — Proposal/implementation mismatch.** Docs and memory describe "cross-project transfer"; code defaults to `scope="project"` which *blocks* transfer. A rule learned 10× in project A is invisible in project B until B independently rediscovers it.
- **C2 — Two independent project-scoping mechanisms.** `SessionStart` injection derives project topics from `rule-activation-index.json` (only rules with `enforce`/`activates_on` blocks). MCP retrieval derives project pertinence from `sources[]` on the rule file. The two sets drift: a topic present in the injection list may yield zero MCP results, and vice versa.
- **W1 — `shared/project/` orphaned from runtime.** Excluded from MCP `_RETRIEVAL_TYPES`; not read by Claude's native auto-memory (which reads `bots/<agent>/memory/` directly). Effectively write-only archive with no formal marker of that fact.
- **W2 — SessionStart injection scales linearly.** Current payload concatenates every vault topic; grows with vault age. No prioritization, no top-N cap, no project awareness when project has no activation-tagged rules.
- **S1 / S2 / S3 (related suggestions).** Auto-promote to universal by `source_count`; unify retrieval through the rule-activation index; make injection a structured protocol instead of a plain sentence.

The retrieval review also left three clarifying questions, answered below as design premises.

### Answers to the clarifying questions (design premises)

1. **Scope semantics.** Rules are **local by default**; a rule becomes **universal** automatically when `distinct_projects(sources) >= N` (default `N = 2`). Universal rules cross projects; local rules stay siloed. Transfer is observable (a threshold crossing) rather than implicit (a filter default).
2. **`shared/project/` role.** Human-viewable surface only. Runtime context for project-specific facts is delivered through Claude's native auto-memory, which already mirrors `bots/<agent>/memory/`. The consolidated `shared/project/<slug>.md` gets an explicit `runtime: false` frontmatter field and a documentation note. No retrieval changes.
3. **`primacy@5` denominator.** Stays project-scoped (matches the retrieval default, which now means local + universal). A new secondary metric `universal_surface@N` measures how often a universal rule is the top retrieved match when queried from a project that has no local copy.

---

## Goals

- Eliminate the scope-semantics gap between documentation and behaviour (C1).
- Collapse the two project-scoping mechanisms into one derived index (C2).
- Formalize `shared/project/` as a human surface (W1).
- Make SessionStart injection bounded, structured, and project-aware (W2, S3).
- Auto-promote rules to universal by observation, with no manual curation (S1).
- Make MCP retrieval read from the unified index rather than re-globbing the filesystem (S2).

## Non-goals

- Changing the LLM extraction pipeline or consolidation contract (**W3** — deferred to v0.8.0).
- Adding UI, visualization, or Obsidian plugin integrations (violates the "injection engine, not visualizer" principle).
- Allowing manual `universal: true` edits in frontmatter (would reintroduce manual curation).
- Introducing hierarchical scopes like `org` or `team` (YAGNI; revisit only when evidence demands it).

## Architecture

One builder produces one index, read by three consumers:

```
                 build_index(vault)
                         │
                         ▼
       .mnemo/rule-activation-index.json  (schema_version: 2)
                         │
        ┌────────────────┼─────────────────┐
        ▼                ▼                 ▼
  SessionStart     MCP tools         PreToolUse
  (topics per     (rules, topics,    (enforce +
   scope, top-N)   bodies)            enrich per project)
```

- **Builder**: `mnemo.core.rule_activation.build_index` (extended). Walks `shared/feedback/*.md` (and extends to the other retrieval types — see "Schema changes"). Gates every file through `is_consumer_visible`.
- **Persistence**: `.mnemo/rule-activation-index.json`, atomic write via existing `_atomic_write_bytes`. Schema bump `INDEX_VERSION = 2`; v1 indexes load as `None` and trigger a rebuild on the next SessionStart.
- **Consumers read only**: `load_index` remains fail-open (returns `None` on any error). All consumers degrade gracefully on a missing or stale index.

### Schema v2

```jsonc
{
  "schema_version": 2,
  "built_at": "2026-04-18T12:34:56Z",
  "vault_root": "/home/.../mnemo",

  "rules": {
    "<slug>": {
      "type": "feedback|user|reference",
      "name": "...",
      "topic_tags": ["code-style", "git"],
      "source_files": ["bots/a/memory/feedback_foo.md", "bots/b/memory/feedback_foo.md"],
      "source_count": 2,
      "projects": ["a", "b"],
      "universal": true,
      "body_preview": "first ~300 chars of body...",
      "enforce":       { "tool": "Bash", "deny_patterns": [...], "deny_commands": [...], "reason": "..." } /* or null */,
      "activates_on":  { "tools": ["Edit"], "path_globs": ["**/*.py"] } /* or null */
    }
  },

  "by_project": {
    "<project>": {
      "local_slugs": ["<slug>", ...],
      "topics": ["code-style", "database", ...]
    }
  },

  "universal": {
    "slugs": ["<slug>", ...],
    "topics": ["git", "workflow", ...]
  },

  "malformed": [{ "path": "...", "error": "..." }]
}
```

Notes on the shape:

- `rules` is the primary table; `by_project` and `universal` are derived lookups to keep read-paths O(1) per query.
- `universal` is derived at build time: `len(projects) >= cfg.scoping.universalThreshold`. Never persisted in frontmatter; `sources[]` is authoritative.
- `enforce` and `activates_on` are embedded per-rule instead of sitting in separate `enforce_by_project` / `enrich_by_project` tables. The match helpers iterate `by_project[<proj>].local_slugs` (and `universal.slugs`) and read the blocks from `rules`. One fewer place for data to diverge.
- `malformed` is unchanged from v1.

### Demotion and re-promotion

`universal` is re-derived on every build. If a user deletes the memory files in project B and the rule's `distinct_projects` falls to 1, the rule reverts to local on the next SessionStart. No migration gymnastics; the index is derived state.

## Component changes

### `core/rule_activation.py`

- `INDEX_VERSION` bumped to `2`.
- `build_index` extended to emit the schema above. The existing `enforce_by_project` / `enrich_by_project` top-level keys are **removed** in v2. The match helpers are updated to iterate `by_project[proj].local_slugs` + `universal.slugs` and read the per-rule blocks from `rules[slug]`.
- New helper: `_is_universal(projects: list[str], threshold: int) -> bool`.
- The builder now walks all three retrieval-eligible types (`feedback`, `user`, `reference`) rather than just `feedback`. Today only feedback has enforce/activates_on, but the unified retrieval index must cover all types MCP can return. `shared/project/` is still excluded (consistent with `_RETRIEVAL_TYPES`).
- `match_bash_enforce(index, project, command)` and `match_path_enrich(index, project, file_path, tool_name)` updated to read the new layout. Behaviour-equivalent for callers.

### `core/mcp/tools.py`

- Retrieval functions read from the index instead of globbing. The previous helper `_rule_belongs_to_project` is replaced by `_rule_in_scope(rule: dict, project: str | None, scope: str)`.
- `scope` values:
  - `"project"` (default): `project in rule.projects` **OR** `rule.universal`.
  - `"local-only"`: `project in rule.projects` (legacy v0.6.2 behaviour).
  - `"vault"`: always True.
- `list_rules_by_topic`, `read_mnemo_rule`, `get_mnemo_topics` all honour the same `scope` vocabulary.
- When the index is missing (e.g. fresh vault before first SessionStart, or load failure), the MCP functions **fall back** to the current glob+parse path so MCP calls keep working — but the fallback pretends every rule is local (universality only applies when the index is available). This keeps the change backward-compatible under degraded conditions.

### `hooks/session_start.py`

- `_build_injection_payload(vault_root, current_project)` rewritten to read the index:
  - Load `by_project[current_project].topics` → `local_topics`.
  - Load `universal.topics` → `universal_topics`.
  - Apply top-N cap from `config.injection.maxTopicsPerScope` (default `15`). Topics ordered by aggregated `source_count` descending (sum across rules carrying that topic). Stable secondary sort by name.
  - Emit the envelope:

    ```
    mnemo://v1 project=<current-project>
    local: [topic, topic, topic]
    universal: [topic, topic]
    Call list_rules_by_topic(topic) then read_mnemo_rule(slug) BEFORE writing code.
    Use scope="project" for local+universal, scope="local-only" to exclude universal.
    ```

  - When `current_project` is `None` or has no local rules, omit the `project=...` token and emit only `universal: [...]`. This is an explicit degraded mode; telemetry counts it.
  - When the index is missing: emit a minimal envelope listing `get_mnemo_topics()` output (vault-wide). Same fail-open spirit as MCP.
- The payload remains wrapped in a defensive try/except — failure here must never block session startup.

### `core/extract/promote.py`

- When writing `shared/project/<slug>.md`, include `runtime: false` in the frontmatter. This is a documentation marker; no code reads it. Its purpose is to make the contract visible in the file itself and in downstream diffs.

### `core/config.py`

- New defaults:
  - `scoping.universalThreshold` → `2`
  - `injection.maxTopicsPerScope` → `15`
- Existing config keys unchanged.

### Docs

- `README.md`: add a "Scope model" section summarizing local-by-default + universal promotion at `distinct_projects >= 2`.
- `templates/HOME.md`: add a one-liner under "Tier 2" clarifying that `shared/project/` is a human navigation surface, not a runtime input.
- `CHANGELOG.md`: v0.7.0 entry listing the breaking semantic change to `scope="project"` and the new `scope="local-only"` for legacy behaviour.

## Scope semantics — API contract

| `scope`       | Returns                                                    | Default | Replaces              |
|---------------|------------------------------------------------------------|---------|-----------------------|
| `"project"`   | rules with `project` in `rule.projects` **OR** `universal` | yes     | v0.6.2 `"project"` (local-only) |
| `"local-only"`| rules with `project` in `rule.projects`                    | no      | new — legacy behaviour |
| `"vault"`     | all consumer-visible rules                                 | no      | unchanged             |

**Breaking change.** Callers that previously relied on `scope="project"` meaning "local only" must migrate to `scope="local-only"` to preserve semantics. The default shift is the fix for C1 and the entire reason for the release.

## Migration

- First SessionStart after upgrade:
  - `load_index` returns `None` because `schema_version` is still `1`.
  - `session_start.py` already rebuilds the index when activation flags are on; v0.7.0 makes the rebuild unconditional when `injection.enabled` is also on. The rebuild is O(rules) and completes in milliseconds for any realistic vault.
- Existing vaults gain universal promotion automatically based on their current `sources[]` state. No user action. No data migration. No file format change in `shared/`.
- `mnemo doctor` gains a new check: **"universal promotion health"** — reports total universal rules, top universal topics, and any rules within 1 project of crossing the threshold (early-warning signal).

## Testing strategy

TDD for every new behaviour. Tests split across unit, integration, and live-recall:

### Unit
- `_is_universal([...], threshold)` — boundary at N.
- `_rule_in_scope(rule, project, scope)` — cartesian of (local | universal | neither) × ("project" | "local-only" | "vault") × (project match | no match).
- Top-N topic selection — cap enforcement, ordering stability, local-before-universal fill.
- Injection envelope shape — snapshot test covering project present, project absent, degraded mode.

### Integration
- `build_index` over a synthetic vault with mixed rule types, source distributions, and malformed files. Assert `rules`, `by_project`, `universal`, `malformed` all contain the expected entries.
- Round-trip: write index v2 → load_index → read back same structure.
- v1 → v2 migration: a v1 index on disk triggers a rebuild on next SessionStart.
- MCP tools hit the index (fast path) and degrade to glob+parse (fallback path) with identical result sets.

### Regression
- Golden-output comparison: MCP calls on v0.6.2 vs v0.7.0 with `scope="local-only"` must return bit-identical slug lists (proves the legacy escape hatch is truly equivalent).

### Live-recall
- Run `mnemo recall` against a real vault before/after upgrade. `primacy@5` must not degrade.
- New metric `universal_surface@5` begins emitting; baseline recorded on first post-upgrade run.

## Rollout

- v0.7.0 ships behind the existing runtime defaults (`injection.enabled`, `enforcement.enabled`, `enrichment.enabled` — all `true` since v0.6.0).
- No feature flag. The breaking scope-default change is the headline of the release.
- Release notes + CHANGELOG call out the `scope="local-only"` escape hatch explicitly for anyone scripting MCP calls directly.
